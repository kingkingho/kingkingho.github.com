总结

CHAR和VARCHAR类型

  • CHAR是定长类型，对多可存储255个字符，utf8字符集下，最大存储85个字符(255/3)。
  • CHAR(32)在utf8下存abcd四个字符，仍然会占用96个字节(32*3)。
  • VARCHAR是变成类型，最大65535字节，由于需要额外1-2个字节做长度标识，单表所有VARCHAR字段列可用总长度为65533字节。
  • VARCHAR在不跨越255字节的情况下，增加列长度可以实现inplace，不需要拷贝(迁移)原数据，只需要修改metadata即可。
  • VARCHAR实际存储长度超过255字节时，会被当做为TEXT处理。

BLOG和TEXT类型

  • 超长字段可能会发生off page(overflow)存储。
  • 大对象类型尽量不要使用，无法避免的情况下单独一张表存放。
  • 对text列排序时，实际排序长度是max_sort_length字节。

数据类型的选择

  • 最大限度减少I/O请求。
  • 让数据表每行长度越小越好。
  • 有TEXT/BLOG等大列时，尽量拆分到独立子表。
  • 经常更新的列和不经常更新的列分开存储。
  • 更新前后的长度也尽可能不要变大。

类型隐式转换

  • 对NULL进行比较时，只能是IS [NOT] NULL，或者<=>NULL，其他情况无法正确计算，也无法走索引。
  • 向字符串列传入整型比较时，整型会被隐式转换成字符串（无法使用索引）。
  • TIMESTAMP/DATETIME列和整型比较时，整型会自动转换成日期时间类型（5.6以后可以使用索引）。
  • 字符串列和0比较时，全部字符串行会匹配上。
